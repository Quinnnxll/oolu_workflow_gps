# Workflow-GPS v0.7.0 — the Unified release

Two arcs land at once. **The planner grows a brain**: models advise picks
as priors that evidence washes out, posteriors learn to forget, cost
becomes a declared trade, every run feeds a training corpus, and planner
changes audition on replayed history before touching money. **The product
grows one surface**: the multi-tenant gateway is now the only shell —
`wfgps desktop` (loopback, auto signed in) and `wfgps host` (network,
real accounts) are two bindings of the same thing, and the CAD pack
proves the whole economy generalizes beyond office work.

## Get it

- **Desktop** — unzip, double-click `setup.bat` (Windows) or run
  `./setup.sh` (macOS/Linux). Your browser opens straight into the shell,
  signed in automatically: on your own machine the loopback bind — not a
  password — is the trust boundary.
- **Self-host for a team** — `wfgps host` (or `docker compose up` with
  `WFGPS_HOST_SECRET` + `WFGPS_ADMIN_PASSWORD`): every person gets a
  username, password, and stored authority. Browsers sign in at `/`; API
  clients at `POST /v1/auth/login`. Terminate TLS in front.
- **Developers** — `pip install -e ".[serve]"`, then `wfgps desktop`;
  run `wfgps doctor` whenever anything misbehaves.

## Highlights

### The planner grows a brain
- **Models advise, evidence decides.** The assembler's new
  `ProposalModel` seam lets a model weigh contested picks — but its
  weights enter the same Beta posterior verified history feeds, as
  pseudo-observations that decide thin-history ties and wash out as real
  runs accumulate. A dead model endpoint degrades to history-only
  assembly; a single candidate never spends a model call. Two
  implementations ship: `GatewayProposalModel` (the routing gateway's
  LLM, cache-safe prompt, defensively parsed) and `TraceProposalModel` —
  the free baseline learned from **your own run log**, wired in by
  default and tenant-scoped, which any future sequence checkpoint
  (Mamba/SSM via the same protocol) must beat in the replay harness to
  earn its inference cost.
- **Model calls are never free.** `billing.model_calls` meters every
  completion's tokens into money; previews surface `planning_cost`, and
  the budget verdict judges **gross + planning cost**.
- **Thompson v2.** `TraceStore(recency_decay=…)` makes posteriors track a
  node's *recent* self — a node that regressed last month stops looking
  as good as ever. `cost_weight` turns picks into expected **utility**
  (quality minus weighted personal cost), so a slightly-less-proven cheap
  node can honestly beat a proven dear one. Previews report
  `expected_success` — the plan's chance of verified success in your
  hands.
- **Strategies audition before they ship.** `knowledge.replay` runs
  planner strategies through seeded worlds (fittable from your recorded
  history) with drift as a first-class phase — reports compare decisions,
  not luck. The tests use it to *prove* decay adapts faster and
  cost-awareness buys success cheaper.
- **Every run becomes training data.** The trace store logs runs
  verbatim; `knowledge.corpus` exports (goal, plan-prefix → next node)
  examples as time-ordered JSONL for offline model training.

### The first domain pack: CAD, verified by mathematics
- `domains.cad`: deterministic OpenSCAD rendering plus a verification
  predicate grounded in exact geometry — divergence-theorem volume
  (hypotheses *asserted in tests*: translation invariance, orientation
  antisymmetry), combinatorial watertightness, Euler characteristic and
  genus (a plate with one bolt hole must be genus 1). A failed predicate
  fails the action → the run → the earnings, and the trace posterior
  records it honestly. The starter pack's spec bounds bracket closed
  forms, recomputed inside the tests; CAD nodes are ordinary marketplace
  citizens.

### One surface, real users
- **Local accounts** (`identity.accounts`): scrypt-hashed passwords,
  short-lived HS256 tokens through the same `OidcValidator` an external
  IdP would use, roles that become **stored grants** (a forged token
  claim buys nothing), uniform login failures (no account enumeration),
  decoy hashing (uniform timing), and brief lockouts.
- **The shell**: sign-in page + screens over the authenticated surface —
  runs with every pause actionable (clarify / confirm route / approve /
  incident / cancel), goal assembly with priced previews and budget
  verdicts, the approvals inbox, skills search, earnings with payout
  onboarding, user management, and health with execution-isolation
  labels. Screens degrade honestly (403 → "no authority", 404 → "not
  enabled on this host"). Real-Chromium tours drive all of it, including
  a run paused twice to completion.
- **`wfgps desktop` = the same gateway, zero ceremony**: a local user is
  auto-provisioned and signed in via a `#auth` bootstrap that moves the
  token into sessionStorage and out of the URL. Ephemeral credentials
  every launch; durable data in one directory.

### Onboarding that cannot dead-end
- **`wfgps doctor`** checks Python, the data dir, each optional stack
  (naming the exact extra), the configured model endpoints, and the
  API-key requirement — every failure with its one-line fix. `wfgps run`
  preflights the same traps instead of tracebacking.
- Bare-file `cli.py` invocation and `uvicorn …asgi:app` answer with
  directions; the setup scripts bootstrap pip into stripped venvs; every
  SQLite store creates its schema **and its parent directory** at
  construction — the packaged app's first-run crash on fresh machines,
  root-caused and pinned.
- A `ci` workflow (lint + full suite, `workflow_dispatch`) joins the
  installer builds; all actions run current Node-24 majors.

## Verification

757 tests: the money-invariant fuzzing, store-contention stress, and
economy suites carry over; new proof-grade suites cover the mesh
mathematics against closed forms, bandit strategies under drift in the
replay harness, identity (uniform failures, lockouts, grants), fresh-
directory startup for every store, and real-Chromium tours of sign-in,
user management, the task flow, and the auto-auth desktop.

## Breaking changes

- **The loopback surface is removed.** The `workflow_gps.desktop`
  package (`DesktopService`, the loopback app and its `/v1/tasks`-style
  routes), `build_desktop_runtime`, and `wfgps web` (with
  `WFGPS_WEB_TOKEN`) are gone. `wfgps desktop` now serves the gateway
  surface (same flags, port, and data layout — setup scripts and the
  packaged app unchanged); network self-hosting is `wfgps host`
  (`WFGPS_HOST_SECRET` + `WFGPS_ADMIN_PASSWORD`), which the Docker image
  now runs.
- `metering.model_calls` moved to **`billing.model_calls`** (metering
  counts usage; billing prices it — the metering package's own tested
  invariant).
- `NodeStats` / `NodePosterior` counts are **floats** (recency-decayed
  observations are fractional).
- Budget verdicts on assembly previews judge **gross + planning_cost**;
  a plan that needed model advice is honestly dearer.
- The trace store gains a raw run log via migration (existing databases
  adopt it cleanly; nothing to do).
